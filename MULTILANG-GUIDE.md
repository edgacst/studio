# 다국어(i18n) 기능 가이드

## 🌏 개요

'지혜로운 손길' 웹사이트는 **4개 언어**를 지원하며, 각 언어별로 **창의적으로 차별화된 디자인**이 적용됩니다.

### 지원 언어
- 🇰🇷 **한국어** (Korean) - 기본 언어
- 🇺🇸 **영어** (English)
- 🇯🇵 **일본어** (Japanese)
- 🇨🇳 **중국어** (Chinese)

## 🎨 언어별 디자인 차별화

### 1. 한국어 테마 (Korean)
- **색상**: 네이비 + 골드 (태극 색상 기반)
- **배경**: 빨강(#C60C30) + 파랑(#003478) 그라데이션
- **느낌**: 전통적이면서도 프리미엄한 느낌

### 2. 영어 테마 (English)
- **색상**: 블루 + 실버 (성조기 색상 기반)
- **배경**: 빨강(#B22234) + 진한 파랑(#3C3B6E) 그라데이션
- **느낌**: 모던하고 비즈니스 전문성

### 3. 일본어 테마 (Japanese)
- **색상**: 체리블라섬 핑크 + 화이트 (일장기 색상 기반)
- **배경**: 빨강(#BC002D) + 화이트 그라데이션
- **느낌**: 우아하고 세련된 느낌

### 4. 중국어 테마 (Chinese)
- **색상**: 레드 + 골드 (오성홍기 색상 기반)
- **배경**: 빨강(#DE2910) + 금색(#FFDE00) 그라데이션
- **느낌**: 럭셔리하고 화려한 느낌

## 🔧 작동 방식

### 언어 선택기
1. **헤더 우측 상단**에 언어 선택 드롭다운 배치
2. **현재 언어**: 국기 아이콘 + 언어명으로 표시
3. **클릭 시**: 드롭다운 메뉴에서 영어/일본어/중국어 선택 가능
4. **호버 효과**: 각 언어별 색상으로 하이라이트

### 자동 기능
- **브라우저 언어 감지**: 첫 방문 시 사용자의 브라우저 언어 자동 인식
- **언어 저장**: 선택한 언어를 `localStorage`에 저장하여 다음 방문 시 유지
- **실시간 변경**: 언어 변경 시 페이지 새로고침 없이 즉시 적용
- **전체 적용**: 모든 텍스트와 디자인이 자동으로 변경

## 📂 파일 구조

```
js/
└── i18n.js          # 다국어 번역 데이터 및 로직
css/
└── style.css        # 언어별 테마 CSS (body.lang-ko, .lang-en, .lang-ja, .lang-zh)
index.html           # data-i18n 속성으로 번역 키 연결
```

## 🔑 번역 키 구조

### HTML에서 사용법
```html
<!-- 텍스트 번역 -->
<h2 data-i18n="services.title">제공 서비스</h2>

<!-- 플레이스홀더 번역 -->
<input type="text" data-i18n-placeholder="contact.name_placeholder">

<!-- 버튼 value 번역 -->
<input type="submit" data-i18n="contact.submit_button">
```

### JS 번역 데이터 구조
```javascript
const translations = {
    ko: {
        nav: { home: '홈', services: '서비스', ... },
        hero: { title1: '당신의 성공을 위한', ... },
        features: { title1: '65년의 경험', ... },
        services: { subtitle: '맞춤형 솔루션', ... },
        // ...
    },
    en: { ... },
    ja: { ... },
    zh: { ... }
};
```

## 🚀 테스트 방법

1. **브라우저에서 `index.html` 열기**
2. **우측 상단 언어 선택기 클릭**
3. **다른 언어 선택** (예: English, 日本語, 中文)
4. **모든 텍스트와 테마 색상이 변경되는지 확인**

### 확인 사항
- ✅ 모든 섹션의 텍스트가 선택한 언어로 변경되는가?
- ✅ Hero 섹션의 배경 그라데이션이 언어별로 다른 색상인가?
- ✅ 드롭다운 메뉴의 호버 효과가 언어별 색상으로 표시되는가?
- ✅ 언어 선택이 `localStorage`에 저장되어 새로고침 후에도 유지되는가?

## 🎯 향후 확장 가능성

### 추가할 수 있는 기능
1. **URL 파라미터**: `?lang=en` 형태로 URL에서 언어 지정
2. **SEO 최적화**: `<html lang="ko">` 속성 자동 변경
3. **RTL 지원**: 아랍어 등 우측에서 좌측 언어 지원
4. **동적 폰트**: 언어별로 다른 폰트 적용 (예: 중국어는 Noto Sans SC)
5. **날짜/시간 형식**: 언어별 로케일에 맞춘 날짜/시간 표시

### 새 언어 추가 방법
1. `js/i18n.js`의 `translations` 객체에 새 언어 키 추가
2. `css/style.css`에 새 언어 테마 CSS 추가
3. `index.html`의 드롭다운에 새 언어 옵션 추가

```javascript
// js/i18n.js에 추가
const translations = {
    // ... 기존 언어들
    fr: {  // 프랑스어 추가 예시
        nav: { home: 'Accueil', services: 'Services', ... },
        // ...
    }
};
```

```css
/* css/style.css에 추가 */
body.lang-fr {
    --theme-accent: #0055A4;  /* 프랑스 국기 색상 */
}

body.lang-fr .hero {
    background: linear-gradient(135deg, rgba(0, 85, 164, 0.03) 0%, rgba(239, 65, 53, 0.03) 100%), var(--gradient-dark);
}
```

## 💡 사용 팁

### 1. 번역 품질 향상
- 현재는 기본 번역이 적용되어 있습니다
- **전문 번역가**의 검토를 권장합니다
- 특히 **마케팅 문구**는 현지화(localization)가 중요합니다

### 2. 이미지 텍스트
- 포트폴리오 이미지에 텍스트가 포함된 경우
- 언어별로 **다른 이미지**를 사용하는 것이 좋습니다

### 3. 성능 최적화
- 현재는 모든 언어 데이터를 한 번에 로드합니다
- 트래픽이 증가하면 **언어별 JSON 파일 분리** 고려

### 4. SEO 최적화
- 각 언어별로 **별도의 URL** (예: `/en/`, `/ja/`, `/zh/`)을 만들면 SEO에 유리
- `hreflang` 태그 추가로 검색엔진 최적화 가능
